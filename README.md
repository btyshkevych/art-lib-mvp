## mod_orders
Реалізований семантичний пошук тексів [наказів МОУ](https://mod.gov.ua/diyalnist/normativno-pravova-baza/type-nakazi) з можливістю подачі короткого змісту наказу з використанням llama3.2.

Загальний спосіб використання файлу:
```
file_name.py [термін пошуку] [кількість результатів - число 1 і більше] [додати короткий зміст від AI - 1 або 0]
```
Реалізація звичайного пошуку:
``` 
query_mod_orders.py закупівлі 5 0 
```
Реалізація пошуку з підготовкаю короткого змісту:
```
query_mod_orders.py "грошове забезпечення" 2 1
```
Для вибірки було завантажено 1000 наказів, близько 250 з них містили текст, інші – були скановані (не бралися в опрацювання). Реалізація семантичного пошуку не показала задовільних результатів. У результатах часто трапляються великі тексти із широкою тематикою, що майже не відповідають параметрам запиту. На мою думку основними причинами цього можуть бути: (1) великий обсяг тексту, що потребуює подальшого поділу на простіші документи; (2) високий вміст канцеляриту та «води» в наказах; (3) брудні дані; (5) абревіатури та специфічна лексика (яка мало представлена в корпусах тексів на основі яких створені алгоритми векторизації). Тому є необхідність для подальшого дослідження способів ефективної векторизації цих текстів.
## bsa
Для того щоб переврити, чи чи можливо з запропонованим стеком задовільнити вимоги проекту, для векторизації був взятий Бойовий статут артилерії, який був поділений на окремі пункти (пункт в середньому до 500 слів). Реалізований семантичний пошук та підсумування усіх результатів пошуку в один текст.
Загальний спосіб використання файлу:
```
file_name.py [термін пошуку] [кількість результатів - число 1 і більше] [додати короткий зміст від AI - 1 або 0]
```
Реалізація звичайного пошуку:
``` 
query_bsa.py рекогносцировка 3 0
```
Реалізація пошуку з підсумуванням результатів:
```
query_bsa.py "переміщення батареї на марші" 3 0
```

##  Загальні питання для подальшого удосконалення проектів:
- Покращення результатів семантичного пошуку. Необхідно доопрацювати очищення, структурування даних, попередню обробку текстів, методи векторизації.
- Підвищення продуктивності під час векторизації даних та роботи LLМ (можливе використання multiprocessing)
- Підбір LLМ, удосконалення промптів.
- Додати блоки try-except для керування винятками та помилками
- Документування, рефакторинг, перевикористання коду
- Безпека й адміністрування БД, бек-апи
- Доопрацювання інерфейсу у формі CLI інструменту або веб API
